# Proposal

## What I’m building
I’m building an API-powered recipe search app that helps users find meals by name and browse recipe results in a clean, visual interface. It is for students, busy people, and anyone who wants quick meal inspiration.

## Which API I’m using
TheMealDB API  
https://www.themealdb.com/api.php

## Why I chose this
I chose this API because food is something people search for all the time, and recipe apps are both useful and visually engaging. I also wanted a project that would let me practice working with real API data, dynamic rendering, and user search input in a way that feels polished and fun to use.

## Core features
- Search for recipes by meal name
- Display recipe cards dynamically with image and title
- Click a recipe to view more details
- Show a loading state while data is being fetched
- Show a friendly error or no-results message when needed

## What I don’t know yet
- How to structure async and await cleanly with fetch()
- How to work with API response data and nested JSON fields
- How to dynamically create and update recipe cards in the DOM
- How to handle loading and error states in a smooth way
